##错误处理 ##try-catch语句 try{ //可能会导致错误的代码 }catch(error){ //错误发生时处理 } 如果错误发生catch语句中包含着一个错误信息对象，所有的浏览器都为这个对象保存着一个错误消息的message属性 ECMA-262还规定了保存一个错误类型的name属性，当前所有的浏览器都支持这个属性 ###finally子句 finally子句一经使用，其代码无论如何都会执行，try语句中的代码能不能正常执行，finally子句都会执行，甚至try语句中包含return语句，finally子句也会执行 ```javascript try{ //可能会导致错误的代码 return 2; }catch(error){ //错误发生时处理 }finally{ return 0; } ``` 只要包含finally子句，那么无论try还是catch语句中的return语句都会被忽略 ###错误类型 ECMA-262定义了下列7个错误类型 + Error，Error是基类型，其他错误都继承自该类型 + EvalError，EvalError类型的错误会在使用eval函数而发生异常时抛出简单来说，就是不把eval()当成函数使用，就会抛出错误 + RangeError，会在数值超出相应范围时触发 + ReferenceError,在找不到对象的情况下触发，object expected错误 + SyntaxError，当把语法错误的javascript字符串传入eval函数时 + TypeError，在变量中保存着意外的类型时，访问不存在的方法时，会触发 + URIError，使用encodeURI()或decodeURI()，而URI格式不正确，就会触发这个错误 ###抛出错误 throw操作符，用于随时抛出自定义的错误，抛出错误时，必须要给throw操作符指定一个值，至于是什么类型的值，没有要求 在遇到throw操作符时，代码会立即停止执行 ##错误事件 没有通过try-catch处理的错误，都会触发window对象的error事件，onerror事件处理程序不会创建event对象，他接受三个参数：错误消息，错误锁在的URL和行号 ##常见的错误类型 错误处理的核心，首先要知道代码里会发生什么错误，由于Javascript是松散类型的，而且也不会验证函数的参数，因此错误只会在代码运行期间出现 + 类型转换错误 + 数据类型错误 + 通信错误
